# Implementation Plan

- [x] 1. pSEO 数据模块
- [x] 1.1 建立 pSEO 页面数据层基础结构
  - 建立 pSEO 数据层，支持两种页面类别（切割方向、场景用例），每个页面包含 SEO 元数据、Hero 内容、场景卡片、操作步骤、FAQ 和相关工具链接
  - 提供按页面标识+语言查询、全量列举、按类别筛选等数据访问能力
  - 未知页面标识查询返回空结果，所有数据在构建时确定，无运行时依赖
  - _Requirements: 1.1, 1.3_

- [x] 1.2 (P) 填充切割方向页数据内容
  - 创建 4 个切割方向页（对半切、水平切、垂直切、等分切）的完整数据，每页包含英文和中文两套独立内容
  - 每页至少 2 个使用场景卡片、至少 3 个 FAQ 条目和至少 3 个操作步骤，所有内容与其他方向页及用例页无重复
  - 每页包含该方向专属的 H1 标题、场景描述、SEO 元数据（title、description、OG）
  - 为每页配置至少 3 个相关工具链接，覆盖其他 pSEO 页面和首页/网格工具
  - 方向页数据独立于用例页数据，两者可分别编写后合并，不存在编辑冲突
  - _Requirements: 1.2, 1.4, 1.5, 4.1, 4.2, 4.3_

- [x] 1.3 (P) 填充场景用例页数据内容
  - 创建 5 个场景用例页（长截图分段 split-long-screenshot、Instagram 切图 split-for-instagram、微信朋友圈切图 split-for-wechat、大图打印切割 split-for-print、替代 PS 切片 no-photoshop-slicer）的完整数据，每页包含英文和中文两套独立内容
  - 每页至少 2 个使用场景卡片、至少 3 个 FAQ 条目和至少 3 个操作步骤，所有内容与其他用例页及方向页无重复
  - Instagram 切图页在 Hero 区下方展示帖子尺寸参考信息（如 1080×1080px）
  - 微信朋友圈切图页英文版仅包含工具基本功能介绍（标题、描述和上传入口），FAQ 和微信特定内容仅在中文版呈现
  - 为每页配置至少 3 个相关工具链接
  - 用例页数据独立于方向页数据，两者可分别编写后合并，不存在编辑冲突
  - _Requirements: 1.2, 1.4, 1.5, 5.1, 5.2, 5.3, 5.4_

- [x] 2. (P) 国际化翻译资源扩展
  - 在英文和中文 messages JSON 中新增 pSEO 落地页的 UI chrome 翻译键：区块标题（场景说明、操作步骤、FAQ、相关工具）、CTA 按钮文本、上传区覆盖文本
  - 新增 Hub 索引页翻译键：页面标题、分类标题（切割方向、使用场景、已有工具）、已有工具（首页分割工具、网格工具）的标题和描述
  - 新增页脚中 Hub 链接的翻译键
  - 所有翻译通过项目现有的 next-intl 翻译系统消费，不在组件中硬编码任何 UI 文本
  - _Requirements: 9.2_

- [x] 3. 落地页 UI 组件
- [x] 3.1 (P) 构建相关工具内链组件
  - 创建相关工具内链区块，接收工具列表和语言参数，水平排列 3-4 个链接卡片
  - 链接自动适配语言前缀（英文无前缀，中文 /zh-CN 前缀）
  - 每个卡片展示工具名称、简短描述和跳转链接，样式与首页特性卡片一致
  - _Requirements: 8.1_

- [x] 3.2 构建落地页模板组件
  - 创建落地页模板，接收页面数据、语言和翻译文本，按顺序渲染：导航栏 → Hero 区（overline + H1 + 副标题 + CTA）→ 上传区 → 场景说明区 → 操作步骤区 → FAQ 区 → 相关工具区 → 页脚
  - 复用首页的导航栏、上传交互区、FAQ 折叠项、CTA 按钮、移动端导航、语言切换器、页脚等现有组件，保持视觉和行为一致性
  - 遵循 Luxury/Editorial 设计风格：衬线标题、无衬线正文、暖白/炭黑/金色调色板、零圆角
  - 场景区使用与首页优势卡片相同的样式模式，操作步骤区使用首页步骤卡片样式模式（大数字 + 标题 + 描述）
  - 用户在 pSEO 页面上传图片后导航到主工具工作区（/workspace），与首页上传行为一致
  - 当页面包含平台参考信息时，在 Hero 区下方渲染参考信息框
  - 导航栏和页脚与首页视觉结构一致，所有 UI 文本通过统一的翻译接口传入，不在组件内部硬编码
  - 依赖 3.1 的相关工具内链组件
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 9.3_

- [x] 4. (P) Hub 索引页
  - 在 /tools 路径创建静态路由页面，在路由解析中优先于同层级的动态工具页路由，支持英文和中文双语
  - 以卡片网格形式按三个类别分组展示所有工具：切割方向（4 个）、使用场景（5 个）、已有工具（首页分割工具、网格工具）
  - 创建工具卡片组件，每张卡片包含工具名称、简短描述和跳转链接，使用顶部边框线 + 悬停高亮样式
  - 已有工具的标题和描述通过 messages JSON 翻译提供，不硬编码
  - Hub 页链接到所有 pSEO 页面，形成 Hub-Spoke 内链结构
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 8.2_

- [x] 5. pSEO 动态路由与 SEO 元数据
  - 创建 pSEO 动态路由页面，在构建时为所有语言 × 所有页面标识组合生成静态页面
  - 为每个页面生成独立的 title tag、meta description、canonical URL、hreflang 替代链接（含 x-default）、Open Graph 和 Twitter Card 元数据
  - 页面渲染落地页模板，访问未知页面标识时返回 404
  - 注入三组 JSON-LD 结构化数据：FAQ 问答（来自 FAQ 配置）、操作步骤指南（来自步骤配置）、软件应用信息（免费设计类应用，价格 0 USD）
  - 动态路由不匹配现有的网格、工作区、工具列表等已有路由
  - 遵循现有的语言前缀策略（英文无前缀，中文 /zh-CN 前缀）
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 7.1, 7.2, 7.3, 7.4, 7.5, 9.1, 10.1_

  - 英文版使用无前缀路径（如 /split-in-half），中文版使用 /zh-CN 前缀
  - 为每个 URL 提供语言替代链接和 x-default（指向英文版）
  - _Requirements: 8.3_

- [x] 6.2 (P) 首页页脚添加 Hub 索引页链接
  - 在首页页脚区块中新增指向 /tools 的链接入口
  - 链接文本通过 messages JSON 翻译提供
  - _Requirements: 8.4_
