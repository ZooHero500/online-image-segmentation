# 实现计划

- [x] 1. 扩展导出工具和 Hook，支持选中状态管理与批量下载
- [x] 1.1 (P) 在 ZIP 导出模块中新增批量下载文件名生成函数
  - 新增辅助函数，接收原始文件名，返回 `{原始文件名}_selected.zip` 格式的文件名
  - 保持现有的全量下载文件名函数不变
  - 为新函数编写单元测试，验证输出格式正确
  - _Requirements: 2.3_

- [x] 1.2 在导出 Hook 中新增选中状态管理和批量下载能力
  - 新增选中键集合状态，使用子图片的行列组合作为唯一标识
  - 实现切换单张选中、全选、取消全选、清除选中四个操作方法
  - 实现批量下载方法：根据选中键过滤当前分割结果，将子集传入现有 ZIP 导出函数，使用批量下载专用文件名
  - 批量下载完成后保持选中状态不变
  - 在清除分割结果时联动清除选中状态
  - 扩展 Hook 返回值，暴露选中键集合和所有新增方法
  - _Requirements: 1.2, 1.3, 1.4, 2.2, 2.3, 2.4, 2.5, 2.6, 3.3, 3.4_

- [x] 2. 扩展结果面板 UI，支持多选交互和批量下载操作
- [x] 2.1 为子图片卡片添加复选框和选中视觉反馈
  - 在每张子图片的缩略图左上角叠加复选框控件
  - 选中时以金色边框高亮显示，未选中时保持默认样式
  - 点击复选框区域切换选中状态，不影响图片预览的点击行为
  - 保持现有的单张下载按钮功能不变
  - _Requirements: 1.1, 1.2, 3.2_

- [x] 2.2 在结果面板中添加选中操作栏
  - 在"下载全部"按钮区域下方新增操作栏，包含全选复选框和"下载选中 (ZIP)"按钮
  - 全选复选框在全部选中时显示勾选状态，再次点击取消全选
  - "下载选中"按钮上标注当前选中数量，如"下载选中 (3)"
  - 未选中任何图片时"下载选中"按钮显示为禁用状态
  - 在面板 Header 区域显示"已选 N/M 张"的实时计数
  - 保持现有的"下载全部 (ZIP)"按钮功能不变
  - _Requirements: 1.3, 1.4, 1.5, 2.1, 2.5, 3.1_

- [x] 3. 集成选中状态到主编辑器并处理生命周期
  - 在主编辑器组件中将 Hook 新增的选中状态和方法通过 props 传递给结果面板
  - 在面板关闭回调中调用清除选中状态方法，确保重新打开时选中状态已重置
  - 在重新生成分割结果时验证选中状态已通过 Hook 联动清除
  - 验证现有的"下载全部"和"单张下载"功能不受影响
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 4. 编写选中状态管理和批量下载的单元测试
  - 测试 Hook 的选中切换：单选切换、全选、取消全选、清除选中
  - 测试批量下载仅将选中子集传入 ZIP 导出函数
  - 测试清除分割结果时联动清除选中状态
  - 测试批量下载完成后选中状态保持不变
