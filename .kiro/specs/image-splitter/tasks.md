# 实现计划

- [x] 1. 项目初始化与基础设施搭建
- [x] 1.1 初始化 Next.js 项目并配置开发环境
  - 使用 Bun 创建 Next.js 15 App Router 项目，启用 TypeScript 严格模式
  - 安装并配置 Tailwind CSS 和 shadcn/ui 基础组件库
  - 配置 `@/` 路径别名
  - 创建全局共享的类型定义（分割线、分割结果、历史记录等核心领域类型）
  - _Requirements: 7.1, 7.5_

- [x] 2. 核心算法层实现
- [x] 2.1 (P) 实现图片裁切引擎
  - 基于 Canvas API 实现图片分割功能，接收原始图片和分割线坐标，输出裁切后的 Blob 数组
  - 以原始分辨率裁切，不缩放不压缩，使用 `drawImage()` 9 参数模式进行精确裁切
  - 保持原始图片格式导出（PNG→PNG, JPG→JPG），WebP 输入统一转为 PNG 导出
  - JPEG 导出 quality 设为 0.92
  - _Requirements: 3.1, 3.6, 3.7_
  - _Contracts: ImageSplitter Service_

- [x] 2.2 (P) 实现 ZIP 打包导出服务
  - 将多个分割结果按命名规则 `{原始文件名}_r{行号}_c{列号}.{格式}` 组织并打包为 ZIP 文件
  - ZIP 文件命名为 `{原始文件名}_split.zip`
  - 通过 `dynamic import()` 按需加载 JSZip，不影响首屏包体积
  - 提供单张子图片下载功能，直接触发浏览器下载
  - _Requirements: 3.3, 3.4, 3.5_
  - _Contracts: ZipExporter Service_

- [x] 2.3 (P) 实现统一存储服务
  - 使用 Dexie.js 4.x 封装 IndexedDB，单表 `history` 统一存储图片 Blob 和元数据
  - Dexie schema 仅索引 `id` 和 `createdAt`，Blob 和缩略图作为普通属性不索引
  - localStorage 用于存储轻量级辅助数据（引导状态、UI 偏好等），统一使用 `img-split:` 前缀命名 key（如 `img-split:guide-dismissed`、`img-split:last-record-id`）
  - 实现历史记录的保存、读取、删除、列表查询功能
  - 删除记录时：从 IndexedDB 删除主数据，同时扫描并清除 localStorage 中以 `img-split:record:{id}:` 为前缀的所有关联 key
  - 提供 `clearRecordLocalData(id: string)` 方法封装 localStorage 清理，避免业务层直接操作 localStorage
  - 实现 50 条上限自动清理（超限时删除最早记录并提示用户），清理时同步清除对应 localStorage 数据
  - 提供存储空间使用量查询能力
  - _Requirements: 6.1, 6.2, 6.3, 6.5, 6.6_
  - _Contracts: StorageService Interface_

- [x] 3. Hooks 层实现
- [x] 3.1 (P) 实现撤销/重做状态管理 Hook
  - 基于状态快照栈模式，每次操作保存完整状态快照
  - 支持撤销（回退到前一个快照）和重做（前进到下一个快照）
  - 执行新操作时自动清空重做栈
  - 最大栈深度默认 50 层
  - _Requirements: 2.9, 2.10_
  - _Contracts: useUndoRedo State_

- [x] 3.2 实现分割线状态管理 Hook
  - 管理分割线的添加（默认在画布中央）、拖拽移动和删除操作
  - 实现磁吸计算：拖拽至距离图片边缘或其他分割线 8px 以内时自动对齐
  - 限制分割线只能在图片范围内移动
  - 单方向分割线数量上限 20 条，达上限时禁用添加
  - 集成撤销/重做 Hook，每次操作自动入栈
  - 依赖 3.1 完成
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.7, 2.8_
  - _Contracts: useSplitLines State_

- [x] 3.3 (P) 实现历史记录管理 Hook
  - 封装存储服务，提供历史记录的保存、加载、删除操作
  - 使用 Dexie `useLiveQuery` 驱动历史记录列表的响应式更新
  - 按时间倒序展示记录，新增/删除时 UI 自动刷新
  - 处理存储空间不足的提示逻辑
  - 依赖 2.3 完成
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 6.6_
  - _Contracts: useHistory State_

- [x] 3.4 (P) 实现图片分割与导出调度 Hook
  - 编排图片分割流程：接收图片和分割线坐标，调用裁切引擎生成子图片
  - 在调用分割前校验：若当前无分割线则阻止操作并提示用户"请先添加分割线"
  - 管理分割状态（进行中/完成）和分割结果列表
  - 分割成功后自动调用历史记录 Hook 保存操作记录（原始图片、分割线位置、生成时间）
  - 提供"下载全部"（调用 ZIP 打包服务）和"单独下载"两种下载方式
  - 依赖 2.1、2.2、3.3 完成
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 3.8, 6.1_
  - _Contracts: useImageExport Service_

- [x] 4. UI 组件层实现
- [x] 4.1 实现图片上传组件
  - 支持拖拽上传（HTML5 Drag & Drop API）和点击选择文件两种方式
  - 仅允许 PNG、JPG、JPEG、WebP 格式，文件大小限制 20MB
  - 格式不支持时显示内联错误提示，文件过大时 Toast 提示并建议压缩
  - 上传成功后回调传递文件信息（File、HTMLImageElement、mimeType）
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5_

- [x] 4.2 实现核心裁剪编辑器组件
  - 安装 react-konva 和 Konva，通过 `dynamic import` + `ssr: false` 加载
  - 使用 react-konva Stage/Layer 渲染画布，图片层和分割线层分离以减少重绘
  - 实现画布等比缩放适配：`scale = Math.min(canvasWidth / imageWidth, canvasHeight / imageHeight)`
  - 集成上传组件和分割线 Hook，支持分割线的可视化拖拽和坐标实时显示
  - 实现分割预览（半透明遮罩或网格方式展示分割后的区域）
  - 工具栏：添加横/纵分割线按钮、生成按钮、下载按钮
  - 键盘快捷键：Ctrl/Cmd+Z 撤销、Ctrl/Cmd+Shift+Z 重做、Delete/Backspace 删除选中分割线
  - 通过 `tabIndex={0}` + `onKeyDown` 在容器 div 上监听键盘事件
  - 支持外部传入初始状态（操作台从历史记录还原）
  - 提供 `onSplitComplete` 和 `onImageUpload` 回调供页面层使用
  - 依赖 3.2、3.4、4.1 完成
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 2.7, 2.8, 2.9, 2.10, 3.8, 7.3, 7.4_
  - _Contracts: SplitEditor Props_

- [x] 4.3 实现分割结果预览与下载组件
  - 网格布局展示所有子图片的预览缩略图
  - 每个缩略图附带单独下载按钮
  - 顶部"下载全部"按钮触发 ZIP 打包下载
  - 展示子图片的行列位置和尺寸信息
  - 依赖 3.4 完成
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 4.4 实现历史记录侧边栏组件
  - 使用历史记录 Hook 获取响应式记录列表
  - 每条记录展示缩略图、文件名和操作时间
  - 点击记录加载对应的图片和分割线状态
  - 支持删除历史记录（通过存储服务级联清除 IndexedDB 主数据和 localStorage 中 `img-split:record:{id}:*` 关联数据）
  - 提供"新建"按钮清空当前画布
  - 依赖 3.3 完成
  - _Requirements: 5.2, 5.3, 5.4, 6.1, 6.3, 6.4_

- [x] 5. 页面层实现
- [x] 5.1 实现 SEO 落地页
  - 作为 Server Component，通过 `metadata` 导出提供 SEO 标题和描述
  - 服务端渲染 SEO 内容区域：功能介绍、使用说明、FAQ 等结构化内容
  - 在 SEO 内容下方嵌入裁剪编辑器组件（通过 dynamic import + ssr: false 客户端加载）
  - 实现引导逻辑：首次分割完成或上传第二张图片时，显示非阻断式 Toast 提示进入操作台
  - 依赖 4.2 完成
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 5.2 实现操作台页面
  - 采用 `'use client'` 模式，左侧历史侧边栏 + 右侧裁剪编辑器的双栏布局
  - 支持直接 URL 访问（`/workspace`），无需从落地页跳转
  - 点击侧边栏历史记录项时，将图片和分割线状态传递给编辑器还原
  - 点击"新建"时清空编辑器并显示上传区域
  - 依赖 4.2、4.4 完成
  - _Requirements: 5.1, 5.2, 5.3, 5.4, 5.5, 5.6_

- [x] 6. 响应式适配与全局体验优化
  - 桌面端提供完整功能，移动端保证基本可用（上传、查看）
  - 确保拖拽操作帧率不低于 30fps（react-konva 分层优化）
  - 统一操作反馈：拖拽视觉提示、按钮 loading 状态、操作成功/失败 Toast
  - 确认所有数据处理在客户端完成，不向服务器发送图片数据
