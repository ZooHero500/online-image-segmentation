# 九宫格切图工具设计方案

## 概述

为 ImgSplit 新增独立的九宫格/宫格切图工具页面 (`/grid`)，面向微信朋友圈、小红书等社交媒体场景。用户上传图片后，通过固定裁切框 + 拖拽图片定位的交互方式，快速将图片切割为 3×3、1×3 或 2×2 的网格切片，一键下载 ZIP。

## 目标用户

- 微信朋友圈九宫格发图用户
- 小红书/Instagram 等社交平台内容创作者
- 需要快速将一张图切成多张等分图片的普通用户

## 设计决策

### 为什么是独立页面而非工作台内集成

| 维度 | 独立页面 `/grid` | 工作台内模式切换 |
|------|------------------|------------------|
| 用户意图 | 明确单一："切成 N 宫格" | 需理解复杂编辑器 |
| 认知负荷 | 低，3 步完成 | 高，需学习标尺/分割线 |
| SEO | 可独立优化"九宫格切图"关键词 | 无法单独优化 |
| 移动端 | 可针对触控优化 | 工作台对移动端不友好 |
| 影响范围 | 不影响现有用户 | 增加工作台复杂度 |

**结论**：独立页面，与现有 `/workspace` 并列。

### 交互模式：固定框 + 拖拽图片

对比了三种方案后选定：

1. ~~裁切框 + 网格预览~~：需要操作裁切框四个角 + 移动，移动端体验差
2. ~~一键切割（全自动）~~：无法调整裁切区域，横图/竖图可能裁掉重要内容
3. **固定框 + 拖拽图片**（选定）：框的形状由网格类型决定（3×3 → 正方形，1×3 → 3:1 矩形），用户拖拽图片在框内移动/缩放来选择展示区域

优势：
- 正方形图片无需调整，直接切割（等同全自动的速度）
- 横图/竖图拖拽定位直觉
- 移动端友好（单指拖拽 + 双指缩放）

## 功能范围

### MVP（本次实现）

- **网格类型**：3×3（9 张）、1×3（3 张横排）、2×2（4 张）
- **图片调整**：拖拽移动 + 滚轮/双指缩放
- **白边间距**：开关控制，默认关闭。开启后每张切片四周添加白色边框
- **下载**：ZIP 打包，文件命名 `grid-1.jpg` ~ `grid-N.jpg`（从左到右、从上到下）
- **响应式**：桌面端三栏布局，移动端单列布局
- **i18n**：中英双语

### 后续扩展（不在本次范围）

- 自定义 N×M 行列
- 更多间距样式（彩色边框、圆角等）
- 社交平台预览模拟（朋友圈效果预览）

### 核心状态 (`use-grid-editor`)

```typescript
interface GridEditorState {
  gridType: '3x3' | '1x3' | '2x2'
  offsetX: number        // 图片水平偏移（px）
  offsetY: number        // 图片垂直偏移（px）
  scale: number          // 图片缩放比例，最小值需保证图片完全覆盖裁切框
  withGap: boolean       // 白边间距开关
}
```

### 缩放约束

- **最小缩放**：图片短边恰好填满裁切框对应边（即图片不能小于裁切框，不允许出现空白区域）
- **最大缩放**：原图尺寸的 3 倍（避免过度放大导致画质差）
- **拖拽边界**：图片边缘不能拖入裁切框内部（clamping），确保裁切框始终被图片完全覆盖

### 裁切框比例

| 网格类型 | 裁切框比例 | 每格比例 |
|----------|-----------|---------|
| 3×3 | 1:1（正方形）| 1:1 |
| 1×3 | 3:1（宽矩形）| 1:1 |
| 2×2 | 1:1（正方形）| 1:1 |

### Canvas 渲染方案

使用**原生 Canvas API** 而非 Konva.js：
- Grid 编辑器交互简单（拖拽 + 缩放），不需要 Konva 的对象模型
- 减少 bundle 大小（Grid 页面不加载 Konva）
- Canvas 仅用于最终切割，编辑器用 CSS/DOM 实现图片定位

### 切割算法 (`grid-splitter.ts`)

**输出分辨率**：裁切框映射到原图的实际像素区域，以原图分辨率切割。例如一张 6000×4000 的横图，3×3 模式裁切框映射到 4000×4000 像素区域，每格输出 1333×1333px。上限遵循现有 `image-splitter.ts` 的 16384×16384 Canvas 限制。

```
输入：HTMLImageElement, GridEditorState, 编辑器显示尺寸
1. 根据编辑器显示尺寸与原图尺寸的比例，将 offsetX/Y/scale 换算为原图坐标系
2. 计算裁切框在原图上的实际像素区域
3. 创建临时 Canvas，尺寸 = 裁切区域的原图像素尺寸（受 16384 上限约束）
4. 将原图对应区域绘制到 Canvas
5. 按网格类型将 Canvas 等分为 N 块
6. 如果 withGap=true，每格内容区域缩小为 94%（四周各留 3% 白色边距），白色填充不透明
7. 每块导出为 Blob（JPEG 92% 质量，PNG 图片保持 PNG 格式）
8. 返回 Blob[] 按序编号
```

### 数据流

```
File → validateFiles() → loadImage() → HTMLImageElement
  → GridEditor（DOM 拖拽/缩放）→ 实时更新 offsetX/Y/scale
  → GridPreview（CSS transform 预览切片效果）
  → 点击下载 → grid-splitter.ts 切割 → zip-exporter.ts 打包 → 下载
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 图片加载失败 | 复用现有 `upload-utils.ts` 的错误提示（toast），留在步骤 1 |
| 不支持的格式/文件过大 | 复用现有验证逻辑，toast 提示 |
| Canvas 内存限制（超大图） | 复用现有 `image-splitter.ts` 的 16384 缩放逻辑，自动降采样 |
| ZIP 生成失败 | toast 提示"下载失败，请重试"，保留当前编辑状态 |
| 移动端单张保存 | iOS Safari 不支持 `<a download>`，改为在新标签页打开图片，提示用户长按保存 |
