# 国际化（i18n）设计方案

## 概述

为在线图片分割工具添加多语言支持，采用 next-intl + 子路径路由方案。

## 语言支持

| 语言 | locale 代码 | 状态 |
|------|------------|------|
| English | `en` | 首期实现（默认语言） |
| 简体中文 | `zh-CN` | 首期实现 |
| 繁體中文 | `zh-TW` | TODO |
| 日本語 | `ja` | TODO |
| 한국어 | `ko` | TODO |

## 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 路由策略 | 子路径 `/en/`、`/zh-CN/` | SEO 友好，每个语言有独立 URL |
| 默认语言 | English (`en`) | 面向国际市场 |
| i18n 库 | next-intl | Next.js App Router 生态最佳，支持 SSR/SSG、中间件路由 |
| 语言检测 | 浏览器 Accept-Language + cookie 持久化 | middleware 自动处理 |
| 语言切换器 | 落地页顶部 + 工作区顶部工具栏 | 两个页面都有切换入口 |
| 翻译风格 | 英文不直译，符合语境和上下文 | 用户要求 |

## 目录结构

```
src/
├── i18n/
│   ├── routing.ts          # locales、defaultLocale、pathnames 配置
│   ├── request.ts          # getRequestConfig - 加载翻译文件
│   └── navigation.ts       # createNavigation - Link, redirect, usePathname
├── messages/
│   ├── en.json             # 英文翻译
│   └── zh-CN.json          # 简体中文
├── middleware.ts            # 语言检测 + 重定向 + cookie 持久化
├── app/
│   └── [locale]/
│       ├── layout.tsx       # html lang 动态设置，NextIntlClientProvider
│       ├── page.tsx         # 落地页
│       └── workspace/
│           └── page.tsx     # 工作区
└── components/              # 位置不变，内部用 useTranslations() 获取翻译
```

## 实现要点

### 1. Middleware

- 检测 `Accept-Language` header，匹配支持的 locale
- 默认 `en`，中文浏览器自动跳转 `/zh-CN/`
- 用户手动切换后 cookie 持久化选择

### 2. 页面迁移

- `src/app/page.tsx` → `src/app/[locale]/page.tsx`
- `src/app/workspace/page.tsx` → `src/app/[locale]/workspace/page.tsx`
- `src/app/layout.tsx` → `src/app/[locale]/layout.tsx`
- `html lang` 属性动态读取 `locale` 参数

### 3. 组件翻译

- 客户端组件使用 `useTranslations('namespace')` hook
- 服务端组件使用 `getTranslations('namespace')`
- 带变量的文本使用 ICU 消息格式：`t('key', { count: 5 })`
- toast 消息同样走翻译

### 4. 语言切换器组件

- 新建 `src/components/LocaleSwitcher.tsx`
- 使用 next-intl 的 `usePathname` + `useRouter` 切换 locale
- 保持当前页面路径不变，仅替换 locale 前缀
- 放置于落地页顶部导航栏和工作区工具栏

### 5. SEO

- `layout.tsx` 中动态设置 `<html lang={locale}>`
- `metadata` 通过 `generateMetadata` 按 locale 生成
- 添加 `hreflang` alternate links

### 6. 日期/数字格式化

- `HistorySidebar` 的 `formatTime` 改用 next-intl 的 `useFormatter`
- 根据 locale 自动适配日期格式
